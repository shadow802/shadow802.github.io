大模型技术-应用开发

发表于 2026-01-12 分类于 AI

前言

    随着模型技术的快速迭代，逐渐达到一个瓶颈期，不论是算力、数据集大小、模型参数数量、推理算法，均达到一个相对平稳的阶段。接下来，需要充分发掘模型的潜力，进行大规模的生产落地，将先进的技术和昂贵的算力成本转换为实际的生产力，提高模型落地业务的能力。

    在这个大背景下，各种指导AI应用大规模落地生产的工程理论、语言框架如雨后的春笋，大量涌现。

    因各项技术相对较新，而且迭代极快，目前还没有形成生产可用的成熟的技术方案。因此本文只是大体列举一下相应的技术点，作为一个思路，具体技术的选型和核心原理，需要继续考证。

基础架构

网关（Gateway）

AI

    目前主流的Nginx和Envoy两大技术栈，分别有很多项目在上面进行AI网关的快速迭代。

API7/ApiSix/Kong

    以Nginx + OpenResty为底层架构，借助于Nginx强大成熟的能力，和Lua插件的高度可扩展，来实现AI网关的代理。以ApiSix为例，实现了模型路由、Token计算、可观测性、请求改写、流式非流式响应体改写、健康检查、负载均衡、Fallback、MCP等能力。    

Gateway API Inference Extension

    Gateway API是云原生领域网关的API规范，目前已经得到众多网关厂商的认可和实现，作为K8s上Ingress的替代。Istio Gateway和Envoy Gateway都在快速成长中。

MCP

    Model Context Protocol，由Anthropic率先提出，为了解决大模型与外部工具之间的通信问题。对大模型的调用、工具的标准化进行了规范地指导，极大增强了模型工具的普适性。

    MCP协议中，主要分为Host、MCP Client、MCP Server三个主要组成部分。Client代理Host对工具的请求，采用标准的MCP协议与Server通信。

    MCP常见的资源类型：resources、prompts、tools。Python中的FastMCP通过@注解来标准不同的类型。

    在MCP Gateway中，至少还需要解决如下问题：

  1. 除了主流工具已经提供的MCP Server，企业中传统的API如何定义MCP Specification，并成为一个标准的MCP Server。

  2. 网关如何发现这些MCP Server，并自动注册并对外提供使用。

  3. Client、Gateway、Server中的安全问题。

    阿里云目前提供了MSE + Nacos的解决方案。

Agent

    Agent Gateway是多智能体实现互相发现、互相使用的基础。网关能解决Agent服务发现、能力注册、安全调用等一些列问题。

    Kgateway提供了MCP和Agent通信的能力，但目前未实际使用。

AI网关、MCP网关、Agent网关，目前正处在快速迭代中，其中Agent网关，由于多智能体的架构尚不明析，很难找到适合的基础架构。

编排（Orchestrator）

    编排是为了解决多智能体架构下，各Agent的调度、执行顺序等问题。目前主流的agent编排主要都是语言框架层，例如LangGraph、crewAI、n8n等，缺乏平台、基础架构级别的编排方案。

ARK

运行时（Sandbox）

    Sandbox主要为智能体提供代码执行环境和可视化操作环境（例如浏览器）。防止一些影响到宿主机安全的危险操作影响到操作系统的其它功能。

Agent Sandbox

   Agent Sandbox支持不同的运行时：gVisor和Kata，提供隔离的环境供智能体执行一些指令和任务。

E2B

应用框架

协议

JSON-RPC

    JSON-RPC 2.0作为MCP和A2A协议的通信格式，是智能体与外部工具和相互通信的基础。其主要的通信字段包括：

  1. jsonrpc，协议的版本，一般为2.0

  2. id，消息的ID，这可以对应相应的请求和响应。

  3. method，方法。

  4. params/result/error等，根据实际场景的所使用的字段。

Streamble HTTP/SSE

    使用HTTP + SSE的传输协议存在如下问题：

  1. 必须维持SSE长连接，这对服务器、中间基础设施（负载均衡、网关）的要求很高。

  2. 一旦断连，重连之后会失去会话的状态。

  3. 存在两个端点，服务器消息只能通过/SSE传递，请求通过/message发送。

    基于此，MCP提出了使用Streamble HTTP来替代传统HTTP + SSE的方式来实现MCP中的通信。

  1. 连接数降低，按需升级SSE。

  2. 支持会话重连，与客户端有明确的会话保持机制。

  3. 统一端点：/MCP。

A2A

    Agent to Agent，是由Google启动，用来描述智能体之间的通信协议。这在多智能体架构中尤其重要。核心要素：

  1. Agent Card，Agent的元数据，用于服务发现和其它Agent与之交互。

  2. 任务管理，用于在长时间任务的异步通信机制，有唯一ID。

  3. Message，用于Agent之间交换各种格式的消息。

微调

    目前语言模型主要分为通用模型和垂域模型，不论是通用模型还是垂域模型，都是无法完全契合具体业务领域的要求的。如果每个模型，尤其是通用模型，在实际生产落地都需要重新训练模型，这巨大的维护成本无疑是企业生产落地的巨大门槛。因此目前业界都在遵循基础模型训练参数的基础上采用一定的微调手段，使之能符合业务的落地要求，并保持不断迭代升级。

Prompt/Context Engineering

    对提示词或者是模型输入的上下文进行调整，规范模型的输出。RAG、Function Call等。

LoRA

    LoRA(Low-Rank Adaptation of LLMs)，即LLMs的低秩适应，是参数高效微调最常用的方法。

    

RLHF

    采用奖励函数等方式，对模型进行有监督的微调，属于对齐微调（Alignment）的一种。

蒸馏（Distillation）

    模型蒸馏，是一种将大型、复杂模型的知识转移到小型、简单模型的技术。DeekSeek提供了大量蒸馏的模型。

量化（Quantization）

    模型量化，是一种模型压缩和加速技术。主要是将高精度浮点数改为低精度数据类型，减少模型的大小和计算量，提高推理速度、效率。

低代码平台

    使用可视化的方式，开发智能体的平台。适合编码能力较低的用户。当然能实现的功能有限，依赖平台的实现。

LangGraph

    LangGraph是一个底层的编排和运行时，专注于智能体在生产上的持久运行。

LangChain

    LangChain是基于LangGraph之上的抽象封装，专注于高层智能体的实现和工具抽象，可快速构建Agent。
